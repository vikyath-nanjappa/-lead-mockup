What this product is — read before writing code
A lead enrichment tool. It takes an existing contact database, finds who is currently in each target role at each company, verifies their email, and surfaces a clean approved list for downstream outreach.
It is not an outreach product. No email sending. No campaign management. Outreach happens externally via Smartlead (Phase 2). Original source data is never overwritten.

F — Developer rules (non-negotiable)
1.TDD first — write tests for edge cases before writing the feature. No feature ships without a passing test.
2.No technical jargon in UI — "catch-all domain" → "Uncertain — can't confirm without sending". "429 rate limit" → "API is busy — pausing briefly". Every error must tell the user what to do next.
3.Never overwrite source data — original_* columns are written once at import and locked. Any write path that touches them is a bug.
4.Credit safety — check available credits before every batch starts, not during. A run that would exceed quota must be blocked before it begins.
5.Midnight safety — calculate estimated completion time before starting. Block the run if it would cross midnight in the configured timezone.
6.Random API delay — every enrichment loop must sleep random(delay_min, delay_max) seconds between calls. Default 3–9s. Never call APIs in a tight loop.
7.One run at a time — if a run is active, the Start button is disabled. No concurrent runs.
8.API keys encrypted at rest — stored in api_credentials table, encrypted. Never in .env files, never in code, never in logs.
9.Background runs — enrichment executes as a background task. UI polls for status. User is notified via email on completion.
10.Settings = single source of truth — no screen other than Settings owns configuration. Other screens display config summaries with "Edit in Settings" links.
